Seamless Wallet API (4.0.0)

Download OpenAPI specification:

Seamless Wallet integration protocol for game providers and operators.

In the Seamless Wallet implementation, there is no wallet on the Provider's side. The wallet exists only on the Operator's side. The Operator is responsible for storing each user's balance and all transaction history.

Roles:

  • Provider — Game provider
  • Operator — Game operation handler

Protocol: JSON over HTTPS POST

Signature

All requests and responses include a signature field — a packet integrity validation hash. The algorithm:

  1. Collect all request/response body fields into a flat key-value map
  2. Add the shared salt (provided during integration onboarding)
  3. Sort the map alphabetically by key (ksort)
  4. Concatenate all values (not keys) into a single string
  5. Compute md5() of the resulting string

PHP example:

$data = [
    "currency"  => "EUR",
    "token"     => "a172bb59a80dde62889950bc289b9640",
    "user_id"   => 123,
    // ... other fields ...
    "salt"      => "*some secret salt*"
];
ksort($data);
$signature = md5(implode('', $data));

Pseudocode (language-agnostic):

fields = { ...request_body, "salt": SHARED_SECRET }
sorted_values = sort_by_key(fields).values()
signature = md5(concat(sorted_values))

Note: The salt key is included in the sorted map for hashing but is never sent in the request/response body. Both sides (Provider and Operator) must use the same salt to produce matching signatures.

Idempotency: Duplicate transaction requests return the original transaction_id and the current user balance.

Response header: All responses contain an internal-request-id header — a unique request identifier for quick transaction lookup on the operator's side.

Provider API

Operator → Provider. Game launch and content delivery endpoints hosted by the Game Provider.

Launch a game session

Opens a game in the provider's iframe/page. Called by the operator's frontend to start a real or demo game session.

The game provider must support auto-resize of game content and mobile games.

query Parameters
action
required
string
Enum: "real" "demo_play"

Game mode:

  • real — real-money game
  • demo_play — demo game
required
integer or string
Examples:

  • game_id=Valhalla - Game by name

Game id (numeric) or absolute game name

required

URL for redirection to casino lobby when the player clicks Home

client_id
string

string <= 2 characters

User's language code (ISO 639-1)

Create a free-spin offer for a player

Creates a free-spin offer on the Provider's side. The Operator sends offer details (game, spin count, bet value, expiration) along with the free_spin_offer_id that will be used to reference this offer in all subsequent wallet callbacks and cancellation requests.

Request Body schema: application/json
required
user_id
required
integer <int64>

Unique player identifier on Operator's side

game_id
required
string <= 255 characters

Identifier of the game

spin_count
required
integer >= 1

Number of free-spins in the offer

bet_value
required
string <decimal> ^\d+\.\d{1,8}$

Bet value per spin

currency
required
string (Currency) <= 10 characters

Currency code. ISO 4217 for fiat (3 chars), or a common ticker for cryptocurrencies (e.g. USDT, BTC).

free_spin_offer_id
required
string <uuid>

Free-spin offer identifier (UUID) assigned by the Operator. Referenced in /free-spin-bet, /free-spin-win callbacks and /cancel-free-spins.

expires_at
required
string <date-time>

Offer expiration date-time (ISO 8601)

signature
required
string (Signature) <= 32 characters

MD5 hash of all packet fields sorted alphabetically (ksort) and concatenated with the shared salt.

Place a win

Credits the win amount to the player's balance.

Important rules:

  • Win with amount = 0 is required for lost bets.
  • Wins without corresponding bets will not be processed.
  • Wins must be repeated until a response with success is received.

string <= 255 characters

Unique identifier of the transaction

bet_transaction_id
required
string <= 255 characters

Transaction ID of the original bet being settled

amount
required
string <decimal> ^\d+\.\d{1,8}$

Gross win amount before tax (0 for lost bets, > 0 for wins)

currency
required
string (Currency) <= 10 characters

Currency code. ISO 4217 for fiat (3 chars), or a common ticker for cryptocurrencies (e.g. USDT, BTC).

Array of objects (Taxes) non-empty

List of taxes calculated by the Provider. Optional — included only when the Provider applies taxes to the win amount on behalf of the Operator. When present, amount in the parent request is always the gross (pre-tax) value. Multiple tax entries support jurisdictions with cascading or composite tax rules (e.g. UA: PDFO + military levy).

signature
required
